All web applications have to load their data from some APIs. The traditional way is using REST APIs.

The 2019 way is using GraphQL

GraphQL is a whole topic and presentation in itself but here are the gists of it:

Schema 
type Project {
  name: String
  tagline: String
  contributors: [User]
}
Query 
{
  project(name: "GraphQL") {
    tagline
  }
}
Result 
{
  "project": {
    "tagline": "A query language for APIs"
  }
}

The server provides a Schema. The schema is typed. The client can query data from the schema and other than with REST only loads the data it wants to use.

In a traditional app you maybe have 3 REST API calls for a specific route. In the GraphQL server and scheam you can bundle these requests into one. While doing that GraphQL provides you ways to cache the data.
