CSS in JS is one of the new next best things. Instead of writing your CSS in CSS, or SASS files you write it directly inside your JavaScript files inside your components.

I think it's nice to have the CSS directly visible in your Components. But where they shine is that it is way easier to react to the current props of the component than with classNames.

Since it's JavaScript you can also use variables and generate stuff like you can with a CSS pre-processor like SASS


styled components

emotion

linaria
styled-system

But we chose not to use CSS in JS. I really wanted to and am still a fan of this approach but it does not make sense for us. We are heavily invested in Bootstrap and SASS. It would not be feasible for us to port everything to JavaScript.

If you start on a blank canvas and if you write your own CSS and do not depend on a UI library I recommend going with a CSS in JS library

styled-components and emotion are evaluated at runtime and linaria is pre-compiled. Out of all styled-components is the most popular

styled-system is a CSS in JS utility library compatible with all 3 libraries
