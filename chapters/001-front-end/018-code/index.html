TypeScript

Flow

There is a huge trend in 2019 for almost every library, framework and larger project to switch to TypeScript. If you haven't heard about TypeScript or Flow they are supersets of JavaScript which also provide type systems

I recommend TypeScript for applications developed by Teams

The Types really help to reduce bugs. They also helpful by describing the interface for your components and thanks to great editor support they will give you auto completion and instant feedback on errors.

ESlint

A static code analysis that checks your code for possible errors or checks against code quality guidelines. It can be dangerous. I've seen project in which rules have been blindly activated before checking what they'll do. And that lead to confusion about React and worse code in general. My advice: Keep ESLint in its default settings for create-react-app.
