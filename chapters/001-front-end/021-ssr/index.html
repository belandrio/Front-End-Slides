React is primarily run on the client at run time. Static or server rendered pages are still faster but single page applications are perfectly fine, unless... you depend on your page being indexable by search engine bots. And those may not be able to run your javascript code before indexing the page. Apparently the GoogleBot is able to run the JavaScript code first but you'll never know how much of it and what about the others?

So SSR to the rescue
